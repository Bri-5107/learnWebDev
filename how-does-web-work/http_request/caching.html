HTTP Caching

Purpose

HTTP Caching allows browser to load resources faster by keeping them in the browser's local storage (cache). This prevents the browswer to repeatedly ask the server for the resource. The only time it requests from the server is when the resource is outdated or missing. The following headers are used in HTTP Caching Messages:
Response Headers:
  1. Cache-Control: "Hi browswer, the resource is fresh for only 10 seconds. After that. requet back to me."
  2. Entity Tag (ETag): "Hey browswer, I am the hashed version of the specific version of the resource. If the resource get updated, I too will be."
  3. Last-Modified: "Hi browser, I contain the date and time when the resource was last modified."
Request Headers:
  1. If-None-Modified: "Hi server, 10 seconds is up, may I ask if the ETag you gave me earlier is still the same as now?"
  2. If-Modified-Since: "Hey server, I want to also ask if the date and time was updated? "
The server returns a 304 NOT MODIFIED if the resource has not been updated.
