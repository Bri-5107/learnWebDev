HTTP Negotation

What?

Problem:
"The website is in Japanese, I can't understand it"
Solution:
HTTP Negotiation allows the client (browser) to demand its preferred resource representation such as:
  • Language (Accept-Language) - English (en), Spanish (es), Japanese (ja)
  • Content-type (Accept) - HTML, XML, JSON
  • Compression (Accept-Encoding) - Gzp, Brotli, Deflate
Then, the server checks if it has the representation the client is requesting. If match, it returns English version of the page, otherwise it will only return what the server has. Furthermore, HTTP Compresssion is an essential characteristic of HTTP requests in where files are compressed into smaller size before sending over the network. This is crucial as it provides:
  • Faster page loading
  • Search Engine Optimization (due to fast page loading)
  • Decrease bandwidth usage
